## 🌐 סקירה כללית

**Navigate** היא מערכת מקצה-לקצה לניהול אימוני ניווט צבאיים.

### יכולות מרכזיות:

- **ניהול מלא** של ניווטים — מהכנה ועד תחקיר
- **מעקב GPS בזמן אמת** אחרי כל מנווט
- **מפות אינטראקטיביות** עם שכבות מידע מרובות
- **עבודה אופליין מלאה** — סנכרון דו-כיווני
- **חלוקת מסלולים** אוטומטית וידנית
- **מערכת ציונים** — אוטומטית ובקרה ידנית
- **תקשורת קולית** (PTT) בזמן אמת
- **לוח בקרה** לסטטיסטיקות ומעקב

## 🏗️ ניהול יחידות היררכי

### מבנה היררכי:

- **יחידות מקוננות** — ללא הגבלת עומק
- כל יחידה: שם, רמה (level), סוג (מנווטים/כללי)
- יחידות מנווטים — מכילות מנווטים פעילים

### תתי-מסגרות קבועות:

- **"מפקדים ומנהלת"** — תמיד קיימת
- **"חיילים"** — ברמת מחלקה (5) ומעלה
- לא ניתנות למחיקה כל עוד היחידה קיימת

### מחיקה מדורגת (Cascade):

- מחיקת יחידה → מחיקת יחידות-בת, עצים, ניווטים
- איפוס משתמשים — יחידה=null, אישור=false

## 🌳 עצי ניווט

### מהו עץ ניווט?

- **קבוצת משתתפים** מאורגנת לניווט
- מכיל **תתי-מסגרות** עם הקצאת משתמשים
- משויך ליחידה ספציפית

### יצירה וניהול:

- יצירת עץ ידנית או שכפול מעץ קיים
- **ייבוא מ-Excel** — טעינת רשימות משתתפים
- הקצאת משתמשים לתתי-מסגרות בגרירה
- הרשאות: "אם רואים — אפשר לערוך"

### כלל חשוב:

> מנהל יחידת-על יכול לערוך תתי-מסגרות של יחידות משנה

## 🔄 מחזור חיי ניווט

### 8 סטטוסים — מהכנה ועד תחקיר:

```
הכנה ← מוכן ← למידה ← בדיקת מערכות ← המתנה ← פעיל ← אישור ← תחקיר
 preparation  ready   learning  system_check   waiting  active  approval  review
```

### קיבוץ במסך ניווטים:

| קבוצה | סטטוסים |
|--------|---------|
| **הכנות ולמידה** | הכנה, מוכן, למידה, בדיקת מערכות |
| **אימון** | המתנה, פעיל |
| **תחקור** | אישור, תחקיר |

> כל תת-מסגרת רואה רק את הניווטים הרלוונטיים לה

## ➕ יצירת ניווט חדש

### 7 חלקים להגדרה:

1. **שטח ומשתתפים** — עץ, תת-מסגרת, גבול גזרה (חובה), סוג ניווט
2. **נקודות** — נ"צ, גבולות, נקודות בטיחות
3. **למידה** — הצגת מסלולים, עריכה, הקראה, PDF
4. **ניווט** — מרווח GPS (5-120 שניות), מקורות מיקום
5. **תקשורת** — PTT, הגדרות ווקי טוקי
6. **תחקיר** — הצגת ציונים לאחר אישור
7. **תצוגה** — הצגת מסלול על מפה

### סוגי ניווט:

- ניווט רגיל | ניווט לילה | ניווט בדד

## 🎙️ ווקי טוקי (PTT)

### Push-To-Talk — תקשורת קולית:

- **לחיצה ודיבור** — הקלטה עד 30 שניות
- **שידור (Broadcast)** — הודעה לכל המנווטים
- **ממוקד (Targeted)** — הודעה למנווט ספציפי

### מאפיינים טכניים:

- הקלטה: `record` package (M4A)
- ניגון: `audioplayers` package
- אחסון: Firebase Storage (`voice_messages/`)
- מטאדאטה: Firestore (`rooms/{navId}/messages/`)
- תור ניגון אוטומטי — הודעות חדשות מתנגנות ברצף

### עקיפות:

- **פר-מנווט**: אפשר/חסום PTT (`overrideWalkieTalkieEnabled`)
- **פר-ניווט**: הגדרה ב-CommunicationSettings

## 📶 עבודה אופליין

### Offline-First Architecture:

- כל פעולה עובדת **ללא חיבור לאינטרנט**
- מסד נתונים מקומי: **Drift (SQLite)** — 17 טבלאות, סכמה v28
- סנכרון דו-כיווני: **SyncManager** ↔ Cloud Firestore

### מנגנון סנכרון:

```
מקומי (Drift/SQLite) ←→ SyncManager ←→ Cloud Firestore
```

- פעולות מקומיות תמיד מצליחות
- סנכרון כשיש רשת — אוטומטי
- Scoped listeners — כל משתמש רואה רק את המידע הרלוונטי

### מפות אופליין:

- הורדת אריחי מפה מראש
- עבודה מלאה עם מפה ללא אינטרנט
